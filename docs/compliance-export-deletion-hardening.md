# Compliance Export and Deletion Hardening

Picom exposes conservative backend-authorized export and staged account-deletion implementations. Hosted deployment, final legal/privacy approval and destructive-worker operations remain gated; the renderer never performs final deletion.

## Data export safety rules

- Placeholder exports must not include passwords, password hashes, session tokens, cookies, privileged Supabase keys, LiveKit credentials, signing keys, or private server fields.
- Placeholder exports must not include private channel data the current user cannot access.
- User-facing text fields are trimmed, control characters are removed, and exported placeholder strings are length-limited.
- The export payload marks itself as renderer-generated and requiring backend verification.
- Production exports should be generated by an authenticated backend job that checks user identity, permissions, and data boundaries.

## Account deletion safety rules

- No hard deletion is performed by the renderer.
- Mock mode stores local request state for UX testing; Supabase mode creates a user-owned backend request, globally revokes sessions, and observes a 14-day grace period.
- Production deletion must run on the backend, revoke sessions, preserve audit log integrity, and handle owned communities safely.
- Owned communities should require transfer, archival, or a reviewed owner-deletion policy before any destructive account action.
- Deleted users should be anonymized or soft-deleted until the final retention policy is approved.

## Logging and diagnostics

- Logs and diagnostics must remain redacted through `loggingService`.
- Passwords, tokens, cookies, secrets, privileged keys, and auth headers must never be written to logs.
- Diagnostics exports should summarize privacy workflow state without embedding credentials or private content.

## Remaining risks

- The current renderer-only export is not a legal/compliance-ready production export.
- Backend verification is still required before users can receive complete account data.
- Backend account deletion confirmation, session revocation, community ownership handling, and audit/account events still need production implementation.
- Legal review is required before final retention, export, and deletion copy is shipped.

## Manual verification

1. Open Settings > Privacy & Safety.
2. Request a data export placeholder.
3. Export the placeholder JSON and confirm it contains only safe profile/settings-style fields.
4. Start account deletion and confirm the UI requires explicit confirmation.
5. Confirm the deletion state says no data has been deleted by the renderer.
