# User Data Export Placeholder

Picom prepares a privacy-safe user data export workflow without generating a full production export in the renderer.

## Current behavior

- Settings > Account includes a Data export section.
- The user can request a local placeholder export.
- The user can download a JSON placeholder with non-sensitive profile/settings summary fields.
- The placeholder explicitly excludes passwords, password hashes, session tokens, authorization headers, private server fields, and internal secrets.

## Future production requirements

- Data exports should be generated by a trusted backend job or Supabase Edge Function.
- Users can only export their own data.
- Private channel data must respect access boundaries.
- Attachments should be represented by safe metadata unless direct file export is intentionally supported.
- Export status should be tracked server-side with expiration and audit/account activity events.

## Manual verification

1. Open Settings > Account.
2. Click Request data export placeholder.
3. Click Download export JSON placeholder.
4. Confirm the file contains only safe placeholder fields and excludes credentials or tokens.
