# Picom Diagnostics and Support Data Notice

Document status: **Engineering draft - legal approval required**  
Draft version: `diagnostics-draft-2026-07-10`

Picom can generate local diagnostic and support exports containing app version, platform, timestamped redacted logs, feature/service states, and error codes. Diagnostics must not intentionally include passwords, session tokens, authorization headers, private keys, Supabase service-role keys, LiveKit secrets, raw message content, captured screen content, or microphone audio.

Diagnostic export is user initiated. Users should review an export before sharing it. Production crash/telemetry providers remain opt-in or disabled until the approved privacy notice, retention period, processor list, regional handling, deletion path, and support contact are published.

This draft does not constitute legal advice, consent language, or a compliance claim. The final document requires named legal/product approval and must match the in-app policy version.
