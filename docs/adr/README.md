# Architecture Decision Records

This directory records major Picom architecture decisions. ADRs should be added when a decision affects runtime architecture, security boundaries, backend strategy, packaging, storage, realtime, permissions, or long-term maintainability.

## ADR template

- Title
- Status: proposed, accepted, deprecated, or superseded
- Context
- Decision
- Consequences
- Alternatives considered
