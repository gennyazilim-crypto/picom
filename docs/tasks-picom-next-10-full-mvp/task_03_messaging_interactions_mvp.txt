EMOJI PICKER
- Opens from MessageComposer emoji button.
- Opens from MessageItem react action.
- Desktop popover style.
- Categories:
  - Frequently Used
  - Smileys
  - Gestures
  - Objects
  - Symbols
- Search input.
- Built-in MVP emoji list.
- Clicking emoji from composer inserts into input.
- Clicking emoji from message adds reaction.
- Close on Escape/outside click.
- Stay within viewport.
- Support light/dark mode.

REACTIONS
- Add reaction to message.
- Remove own reaction by clicking existing reaction.
- Prevent duplicate same emoji by same user.
- Show emoji and count.
- If current user reacted, show subtle active state.
- Use local/mock state.
- Realtime/Supabase connection can be TODO if not already present.

REPLY SYSTEM
Data:
- Add replyToMessageId to local message type if not present.
- Reply target must be in same channel.
- Store reply metadata enough for preview.

Composer:
- Clicking Reply on a message sets reply target.
- Composer shows reply preview:
  - author name
  - short message preview
  - cancel button
- Sending includes replyToMessageId.
- Cancel clears reply target.

MessageItem:
- If message is reply, show MessageReplyPreview:
  - “Replying to author”
  - target preview
- Clicking reply preview can scroll/highlight target if present.
- If target deleted/unavailable:
  - show “Original message was deleted” safe fallback.

EDIT MESSAGE
- Edit own message only.
- Inline editor or modal.
- Enter saves.
- Escape cancels.
- Shift+Enter newline.
- Shows edited indicator.
- Empty edited message blocked unless attachments remain.

DELETE MESSAGE
- Delete own message.
- Moderator/admin/owner can delete others if permission helper exists.
- Show confirmation for delete.
- Use soft deleted placeholder or remove consistently with existing design.
- Deleted message reply targets must show fallback.
