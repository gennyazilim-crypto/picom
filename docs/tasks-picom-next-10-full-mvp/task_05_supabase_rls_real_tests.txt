- docs/task-checkpoints/task-supabase-rls-real-tests.md

TEST SCENARIOS
1. Anonymous user
- cannot read private data
- can read public community metadata only if policy allows
- cannot insert messages
- cannot join community without auth

2. Visitor / non-member authenticated user
- can read public community if visibility public
- can read public non-private channels if public_read_enabled true
- can read messages only in public non-private channels if allowed
- cannot read private channels
- cannot send messages
- cannot upload attachments
- cannot react
- cannot access private attachments

3. Member
- can read joined community
- can read permitted channels
- can send messages in allowed channels
- can react if allowed
- can upload images if allowed
- cannot manage channels unless permission

4. Moderator
- member permissions
- can delete/moderate messages if permission allows
- cannot manage owner/admin-only settings

5. Admin
- can manage channels/members/roles according to permission
- cannot transfer ownership unless owner

6. Owner
- full community access
- can manage community and roles
- can delete/transfer placeholder if implemented

7. Private channel
- hidden from unauthorized users
- messages not visible
- attachments not visible
- mention feed must not leak private content

8. Message ownership
- author can edit own message
- other member cannot edit
- mod/admin can delete if permission allows

9. Storage policies
- attachment read follows message/channel visibility
- upload allowed only for permitted senders
- invalid access denied
